## 🚀 Avancées IA & opportunités à saisir

**L'architecture *Deterministic Core* pour des IA auditées et cohérentes**
Un nouveau pattern d'architecture propose de résoudre les problèmes de cohérence et d'audit des modèles IA en production en ancrant les décisions à un standard fixe via des primitives persistantes. Le papier identifie la statelessness des LLMs comme cause racine des boucles d'audit infinies et des incohérences entre modèles. L'implémentation de référence et une taxonomie des primitives sont documentées, avec des résultats concrets sur des tâches de critique de code. Cette approche pourrait transformer les workflows critiques où la traçabilité est indispensable.

[Lire le papier](https://brandonbellsystems.com/deterministic-core/)

**Gemma 4 QAT : des modèles légers pour l'exécution locale**
Google publie des modèles Gemma 4 optimisés avec Quantization-Aware Training (QAT) pour une exécution locale sur appareils mobiles et laptops. Les améliorations incluent une réduction de la mémoire à 1 Go pour le modèle E2B et des formats de quantification spécialisés pour mobile. Les checkpoints QAT pour Q4_0 et des formats mobiles sont disponibles sur Hugging Face, avec des intégrations pour llama.cpp et vLLM. L'objectif est de préserver la qualité tout en réduisant l'empreinte mémoire et les coûts de calcul.

[Lire l'annonce](https://blog.google/innovation-and-ai/technology/developers-tools/quantization-aware-training-gemma-4/)

**Sidekick : un agent de codage intégré à macOS**
Sidekick est une application macOS en menu bar qui permet d'accéder rapidement à un agent de codage basé sur zot. L'outil propose une interface de chat avec support d'images, gestion de sessions et répertoire de travail configurable. Il utilise les API d'Anthropic, OpenAI ou autres, avec une installation simplifiée via Xcode. Le projet cible les développeurs cherchant une solution légère et intégrée pour le coding assisté par IA.

[Voir le projet](https://github.com/patriceckhart/zot-sidekick)

**Apple Contacts MCP : accès local aux contacts macOS pour les agents IA**
Apple Contacts MCP est un serveur MCP open source pour accéder localement aux contacts macOS via des agents IA. Il permet des opérations sécurisées avec des confirmations dry-run par défaut pour éviter les modifications accidentelles. Le serveur utilise AppleScript pour une intégration native avec Contacts.app et macOS, sans dépendre du cloud. Il est conçu pour des workflows CRM ou de gestion de données personnelles, avec une approche axée sur la confidentialité.

[Voir le projet](https://github.com/lu-wo/apple-contacts-mcp)

**Omni : recherche sémantique locale et privée sur macOS**
Omni est une application macOS (Apple Silicon) permettant une recherche sémantique locale et privée de tous les fichiers via un espace vectoriel unifié. Le moteur utilise un port MLX-Swift de jina-embeddings-v5-omni exécuté en local sur GPU. L'outil propose des vues natives et expose un endpoint local pour les agents IA. Le modèle est téléchargé une fois et fonctionne ensuite sans dépendance cloud, avec un stockage vectoriel SQLite.

[Voir le projet](https://hanxiao.io/omni/)

**Thousand Token Wood : une économie multi-agents sur un modèle 3B**
*Thousand Token Wood* est une simulation économique en temps réel impliquant cinq agents exécutés sur un modèle Qwen2.5-3B. Le projet démontre qu'un modèle de 3 milliards de paramètres peut générer des comportements émergents via des mécanismes de rareté conçus. Les enseignements clés incluent la fiabilité des petits modèles pour la génération de formats mais leur faiblesse en raisonnement économique, corrigée par des prompts optimisés.

[Lire l'article](https://huggingface.co/blog/build-small-hackathon/thousand-token-wood-sim)

**pg_durable : exécution durable dans PostgreSQL**
Microsoft open source **pg_durable**, une extension PostgreSQL permettant d'exécuter des fonctions SQL durables et tolérantes aux pannes directement dans la base de données. L'outil élimine le besoin d'orchestrateurs externes en intégrant la gestion des workflows et des checkpoints dans SQL. Les cas d'usage incluent les pipelines d'embeddings vectoriels et les tâches de maintenance planifiées.

[Voir le projet](https://github.com/microsoft/pg_durable)

## 🛠️ Tech & industrie

**Lockdown Mode d'OpenAI : sécurité renforcée pour les LLMs**
OpenAI introduit *Lockdown Mode*, une fonctionnalité permettant de restreindre les capacités des modèles d'IA pour limiter les risques de jailbreak ou d'utilisation malveillante. Cette option désactive certaines fonctionnalités avancées comme les plugins ou les appels d'API externes, renforçant la sécurité dans des contextes sensibles.

[Lire la documentation](https://help.openai.com/en/articles/20001061-lockdown-mode)

**Google paie SpaceX 920 millions de dollars par mois pour du calcul**
Google a signé un accord avec SpaceX pour louer 110 000 GPU et CPU à 920 millions de dollars par mois entre octobre 2026 et juin 2029. Ce contrat reflète la course aux infrastructures cloud et la dépendance croissante des géants tech aux capacités de calcul externalisées.

[Lire l'article](https://techcrunch.com/2026/06/05/google-will-pay-spacex-920m-per-month-for-compute/)

**La facture des tokens explose : l'industrie cherche des solutions**
Les entreprises réalisent que leurs budgets IA 2026 sont déjà épuisés en avril 2025, en raison de la consommation massive de tokens liée aux agents autonomes. Des cas extrêmes incluent une facture de 500 millions de dollars pour un seul client. Une nouvelle fondation est créée pour instaurer une discipline financière, tandis que les entreprises tentent de limiter les dépenses via des quotas.

[Lire l'article](https://techcrunch.com/2026/06/05/the-token-bill-comes-due-inside-the-industry-scramble-to-manage-ais-runaway-costs/)

**GOV.UK remplace Stripe par Adyen pour les paiements publics**
Le service GOV.UK Pay du gouvernement britannique remplace Stripe par Adyen pour traiter les paiements des collectivités locales et forces de police. Adyen introduit des options de 'pay by bank' via open banking, évitant la saisie de cartes bancaires.

[Lire l'article](https://www.theregister.com/public-sector/2026/06/04/govuk-goes-dutch-on-payments-as-it-dumps-stripe/5250763)
